# Definition of Done - CampusRide

A User Story is considered "Done" only when:

## Development
- [ ] Code is written in Java and follows course naming conventions
- [ ] Code is committed to the `src/` folder in GitHub

## Testing
- [ ] Unit test written and passes for the story logic
- [ ] Acceptance criteria from `backlog.md` are manually tested
- [ ] No critical bugs found during demo

## Documentation
- [ ] Story is linked in `traceability-matrix.md` to SRS requirement and Test Case
- [ ] README is updated if this changes a core feature

## Review
- [ ] Code reviewed and approved
- [ ] Demoed to "product owner" / lecturer
